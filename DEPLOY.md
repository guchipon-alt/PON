# Запуск PON Social Network в интернете (бесплатно, через Render)

Эта инструкция позволяет запустить твою соцсеть по рабочей ссылке вида
`https://pon-social-network.onrender.com` — бесплатно, без покупки домена.

## Что уже сделано в этом архиве
- Порт теперь берётся из переменной окружения `PORT` (нужно для Render/любого хостинга).
- Добавлен `render.yaml` — Render сам поймёт, как собирать и запускать проект.
- Добавлен `Dockerfile` — на случай деплоя через контейнер (Railway/Fly/любой Docker-хостинг).
- Уже есть health-check `/healthz`, лимит запроса 12MB, защита кода подтверждения, rate-limiting.

---

## Шаги (мышкой, без терминала)

### 1. Залить код на GitHub
- Создай на GitHub пустой репозиторий, например `pon-social-network`.
- Загрузи туда содержимое этого архива (кнопка **Add file -> Upload files** прямо на сайте GitHub,
  перетащи все файлы и папки, нажми **Commit**).

### 2. Завести бесплатный аккаунт на Render
- Зайди на https://render.com -> **Get Started** -> войди через GitHub (бесплатно, карта не нужна).

### 3. Создать сервис
- В Render нажми **New +** -> **Web Service**.
- Выбери свой репозиторий `pon-social-network`.
- Render сам подхватит `render.yaml`. Если спросит вручную:
  - **Build Command:** `npm install && npm run build`
  - **Start Command:** `npm start`
  - **Plan:** Free
  - **Health Check Path:** `/healthz`

### 4. Прописать переменные окружения (Environment)
В разделе **Environment** добавь:
- `NODE_ENV` = `production`
- `GEMINI_API_KEY` = твой ключ Gemini (получить бесплатно: https://aistudio.google.com/apikey).
  Без него ИИ-функции просто перейдут в запасной режим — приложение всё равно работает.
- (Опционально, для реальной отправки писем-кодов) `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`,
  `SMTP_PASS`, `SMTP_FROM`. Без них регистрация в проде не сможет отправить код — см. ниже.

### 5. Нажми Deploy
Render соберёт и запустит проект. Через пару минут получишь рабочую ссылку
вида `https://pon-social-network.onrender.com`.

## Важные предупреждения (честно)
1. **Данные обнуляются при передеплое.** `posts-db.json` лежит на диске, а на бесплатном
   Render диск эфемерный — при каждом новом деплое посты/статы сбросятся. Для теста — ок.
   Для постоянного хранения нужна настоящая БД (например, бесплатный Postgres / Supabase).
2. **Сервис «засыпает».** На free-плане после ~15 минут простоя сервис засыпает и просыпается
   ~30 секунд при следующем заходе. Нормально для старта.
3. **Регистрация по email.** Без SMTP-переменных в проде сервер вернёт ошибку при запросе кода
   (это сделано специально, чтобы код не утекал). Чтобы регистрация работала — пропиши SMTP
   (можно бесплатный, напр. Brevo/SendGrid free tier), либо временно тестируй с `NODE_ENV` не
   равным `production` (тогда код возвращается в ответе — только для теста, НЕ для публичного запуска).
4. **Свой домен (например pon.com)** — это уже платно (покупка у регистратора). Поддомен
   `.onrender.com` бесплатный и работает сразу.

---

## Свой домен (когда захочешь)
1. Купи домен у любого регистратора (Namecheap, Cloudflare, reg.ru).
2. В Render: сервис -> **Settings** -> **Custom Domains** -> добавь свой домен.
3. Render покажет, какую DNS-запись (CNAME) прописать у регистратора. Пропиши её.
4. SSL-сертификат Render выпустит автоматически.
